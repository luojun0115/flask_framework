一对多演练

一对多演练

在项目开发过程中，会遇到很多数据之间一对多关系的情况，比如：

  • 商品分类与商品信息的对应关系
  • 班级与学生的对应关系
  • 省份与城市的关系

所以在开发过程中需要使用 ORM 模型将表与表的一对多关联关系使用代码描述出来 一对多关系描述的关键：将关系字段存储在多的一端中

示例场景

需求分析

  • 开发学生管理系统，能够将学生分配到某个班级中

思路分析

  • 需要创建班级表
  • 需要创建学生表

  • 在添加学生时，指定所对应的班级信息

  • 班级表(class)

主键(id) 名称(name)
1 python修正1期
2 python上海2期
3 python深圳3期
  • 学生表(Student)
主键(id) 学生名(name) 所在班级(classid)
1 郭靖 1
2 黄蓉 1
3 杨过 2
4 小龙女 2
5 令狐冲 3
6 胡非 2
7 韦小宝 3

结果

  • 根据学生找到对应的班级，返回单一对象
  • 根据班级找到对应的学生，返回列表对象

代码演练

  • 在models.py中定义模型类
class Classes(db.Model):
    __tablename__ = "classes"
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(64), unique=True)
    # 对象引用关系，根据多中的外键获取数据
    # 参数Student都是类的名称
    # backref表示学生对象获取班级对象的属性名称
    students = db.relationship('Student', backref='classes')


class Student(db.Model):
    __tablename__ = "students"
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(64), unique=True)
    # 关系字段，在表中会生成外键
    # 外键的参数是classes表的id字段
    class_id = db.Column(db.Integer, db.ForeignKey('classes.id'))
  • 迁移：根据模型类在数据库中创建表

  • 创建d4_one2many.py，定义视图index，代码如下
    • 查询编号为1的学生，输出对应的班级名称
    • 查询编号为2的班级，输出对应的学生姓名
from flask import Flask
from models import db, Classes, Student

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'mysql://root:mysql@localhost:3306/demo3'
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = True
db.init_app(app)


@app.route('/')
def index():
    # 查询编号为1的学生，输出对应的班级名称
    stu = Student.query.get(1)
    # stu.classes是班级对象
    print(stu.classes.name)
    print('---------------')
    # 查询编号为2的班级，输出对应的学生姓名
    classes = Classes.query.get(2)
    # classes.students是学生对象的列表
    for stu in classes.students:
        print(stu.name)

    return 'hello one-to-many'


if __name__ == '__main__':
    app.run(debug=True)
