数据库迁移

数据库迁移

  • 在开发过程中，需要修改数据库模型，而且还要在修改之后更新数据库。最直接的方式就是删除旧表，但这样会丢失数据。
  • 更好的解决办法是使用数据库迁移框架，它可以追踪数据库模式的变化，然后把变动应用到数据库中。
  • 在Flask中可以使用Flask-Migrate扩展，来实现数据迁移。并且集成到Flask-Script中，所有操作通过命令就能完成。
  • 为了导出数据库迁移命令，Flask-Migrate提供了一个MigrateCommand类，可以附加到flask-script的manager对象上。

首先要在虚拟环境中安装Flask-Migrate。

pip install Flask-Migrate==2.1.1

主要配置步骤如下

  • 创建models.py，用于定义模型类，代码如下
from flask_sqlalchemy import SQLAlchemy

db = SQLAlchemy()

class User(db.Model):
    __tablename__ = 'users'
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(64), unique=True, index=True)
    email = db.Column(db.String(64), unique=True)
    password = db.Column(db.String(64))
  • 创建d2_migrate.py文件
  • 引入flask-script中的Manager类，创建对象manager
from flask import Flask
from flask_script import Manager

app = Flask(__name__)
app.debug = True
manager = Manager(app)

if __name__ == '__main__':
    manager.run()
  • 创建数据库连接对象
from models import db

app.config['SQLALCHEMY_DATABASE_URI'] = 'mysql://root:mysql@localhost:3306/demo3'
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = True
db.init_app(app)
  • 扩展数据库命令
from flask_migrate import Migrate,MigrateCommand

#第一个参数是Flask的实例，第二个参数是Sqlalchemy数据库实例
migrate = Migrate(app,db) 

#manager是Flask-Script的实例，这条语句在flask-Script中添加一个db命令
manager.add_command('db',MigrateCommand)
  • 最终代码如下：
from flask import Flask, render_template
from flask_script import Manager
from models import db
from flask_migrate import Migrate, MigrateCommand

app = Flask(__name__)
app.debug = True
manager = Manager(app)

app.config['SQLALCHEMY_DATABASE_URI'] = 'mysql://root:mysql@localhost:3306/demo3'
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = True
db.init_app(app)

# 第一个参数是Flask的实例，第二个参数是Sqlalchemy数据库实例
migrate = Migrate(app, db)
# manager是Flask-Script的实例，这条语句在flask-Script中添加一个db命令
manager.add_command('db', MigrateCommand)

if __name__ == '__main__':
    manager.run()

主要命令步骤如下

  • 初始化迁移库，如下命令在pycharm的终端窗口运行，这个命令会创建migrations文件夹，所有迁移文件都放在里面
python d2_migrate.py db init
  • 创建迁移脚本
python d2_migrate.py db migrate -m '创建用户表'
  • 更新数据库
python d2_migrate.py db upgrade
  • 提示：此时books表被删除了，这没关系，正常开发时都会使用迁移进行创建

-此时，数据库demo3中有表users

  • 查看历史版本
python d2_migrate.py db history
  • 回滚到指定版本
python d2_migrate.py db downgrade 版本号

总结：实际操作顺序如下

  • 1.python 文件 db init
  • 2.python 文件 db migrate -m "版本名(注释)"
  • 3.python 文件 db upgrade 然后观察表结构
  • 4.根据需求修改模型
  • 5.python 文件 db migrate -m"新版本名(注释)"
  • 6.python 文件 db upgrade 然后观察表结构
  • 7.若返回版本,则利用 python 文件 db history查看版本号
  • 8.python 文件 db downgrade(upgrade) 版本号
